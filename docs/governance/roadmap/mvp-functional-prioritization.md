# MVP Functional Story Prioritization

> **Language:** English | [Español](./mvp-functional-prioritization.es.md)

This document defines the ordering logic used to derive the MVP backlog and launch-readiness backlog from the functional story catalog. It is the source referenced by the MVP backlog documents.

## Prioritization Rules

- `P1` stories are the first to be delivered.
- Security, onboarding, and tenant-isolation stories take priority when they block launch readiness.
- Post-MVP work is deferred unless it is required for launch, security, or compliance.
