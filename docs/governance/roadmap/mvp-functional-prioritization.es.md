# Priorizacion de Historias Funcionales para MVP

> **Idioma:** [English](./mvp-functional-prioritization.md) | **Español**

Este documento define la logica de orden usada para derivar el backlog MVP y el backlog de preparacion de lanzamiento a partir del catalogo de historias funcionales. Es la fuente referenciada por los documentos de backlog.

## Reglas de Priorizacion

- Las historias `P1` son las primeras en entregarse.
- Las historias de seguridad, onboarding y aislamiento por tenant tienen prioridad cuando bloquean la preparacion de lanzamiento.
- El trabajo Post-MVP se difiere salvo que sea requerido para lanzamiento, seguridad o cumplimiento.
